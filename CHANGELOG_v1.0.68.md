## 🎯 ВОЗМОЖНОСТИ

### 1. **Автоматическое определение типа прокси**

Чекер автоматически определяет тип прокси по префиксу:
- `http://` → HTTP прокси
- `https://` → HTTPS прокси
- `socks4://` → SOCKS4 прокси
- `socks5://` → SOCKS5 прокси
- Без префикса → HTTP прокси (по умолчанию)

### 2. **Поддержка авторизации**

Все типы прокси поддерживают авторизацию:
```
protocol://username:password@ip:port
```

**Примеры:**
```
http://admin:12345@123.45.67.89:8080
socks5://user:pass@98.76.54.32:1080
```

### 3. **Умная обработка**

- Автоматический парсинг прокси строки
- Валидация формата
- Поддержка разных форматов
- Fallback на HTTP если тип не указан

### 4. **Оптимизация производительности**

- HTTP/HTTPS: используется стандартный aiohttp
- SOCKS4/SOCKS5: используется aiohttp-socks
- Минимальные накладные расходы
- Быстрая работа

## 🔧 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### Новые функции в BaseChecker:

#### 1. `_parse_proxy(proxy: str)`

Парсит прокси строку и определяет тип:

```python
proxy_type, proxy_url = self._parse_proxy("socks5://ip:port")
# proxy_type = "socks5"
# proxy_url = "socks5://ip:port"
```

#### 2. `_create_socks_connector(proxy: str)`

Создает SOCKS connector для aiohttp:

```python
connector = self._create_socks_connector("socks5://ip:port")
# Возвращает ProxyConnector для SOCKS
```

#### 3. Улучшенный `fetch()`

Теперь поддерживает все типы прокси:

```python
# HTTP/HTTPS - стандартный способ
resp = await session.request(..., proxy=proxy_url)

# SOCKS4/SOCKS5 - через connector
connector = self._create_socks_connector(proxy)
async with aiohttp.ClientSession(connector=connector) as socks_session:
    resp = await socks_session.request(...)
```

## 📦 ЗАВИСИМОСТИ

### Новая зависимость: aiohttp-socks

Для работы SOCKS4/SOCKS5 прокси нужна библиотека `aiohttp-socks`.

#### Установка:

**Автоматически:**
```bash
pip install -r requirements.txt
```

**Вручную:**
```bash
pip install aiohttp-socks
```

#### Проверка:
```python
python -c "import aiohttp_socks; print('✅ OK')"
```

### Опциональная зависимость

`aiohttp-socks` - **опциональная** зависимость:
- ✅ Если установлена → SOCKS прокси работают
- ⚠️ Если НЕ установлена → HTTP/HTTPS прокси работают, SOCKS выдаст ошибку с инструкцией

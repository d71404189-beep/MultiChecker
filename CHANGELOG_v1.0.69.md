## 🎯 ВОЗМОЖНОСТИ

### 1. **Загрузка из файла**

Создай файл `proxy.txt`:
```
# Мои прокси
socks5://user1:pass1@123.45.67.89:1080
socks4://98.76.54.32:1080
http://11.22.33.44:8080
https://55.66.77.88:443

# Еще прокси
socks5://user2:pass2@111.222.333.444:1080
```

В поле "Прокси" укажи: `proxy.txt`

**Результат:**
```
✅ Загружено 5 прокси
⚠️ Невалидных прокси: 0
```

### 2. **Автоматическая ротация**

Прокси автоматически ротируются при каждом запросе:

```
Запрос 1 → socks5://user1:pass1@123.45.67.89:1080
Запрос 2 → socks4://98.76.54.32:1080
Запрос 3 → http://11.22.33.44:8080
Запрос 4 → https://55.66.77.88:443
Запрос 5 → socks5://user2:pass2@111.222.333.444:1080
Запрос 6 → socks5://user1:pass1@123.45.67.89:1080 (начало цикла)
```

### 3. **Валидация прокси**

ProxyManager автоматически проверяет формат:

**Валидные:**
```
✅ http://123.45.67.89:8080
✅ socks5://user:pass@123.45.67.89:1080
✅ https://123.45.67.89:443
✅ socks4://123.45.67.89:1080
```

**Невалидные (пропускаются):**
```
❌ 123.45.67.89 (нет порта)
❌ ftp://123.45.67.89:21 (неподдерживаемый протокол)
❌ invalid-proxy (неправильный формат)
```

### 4. **Комментарии в файле**

Можно добавлять комментарии:

```
# Быстрые прокси
socks5://fast1@123.45.67.89:1080
socks5://fast2@98.76.54.32:1080

# Медленные но стабильные
http://slow1@11.22.33.44:8080

# Пустые строки игнорируются

http://slow2@55.66.77.88:8080
```

### 5. **Статистика загрузки**

После загрузки показывается статистика:

```
✅ Загружено 10 прокси
⚠️ Невалидных прокси: 2
```

## 🔧 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### Новый модуль: ProxyManager

**Файл:** `checkers/proxy_manager.py`

#### Основные функции:

##### 1. `load_proxies(proxy_input: str)`

Загружает прокси из строки или файла:

```python
# Один прокси
load_proxies("socks5://123.45.67.89:1080")

# Из файла
load_proxies("proxy.txt")
```

##### 2. `get_next_proxy()`

Возвращает следующий прокси (с ротацией):

```python
proxy1 = get_next_proxy()  # Первый прокси
proxy2 = get_next_proxy()  # Второй прокси
proxy3 = get_next_proxy()  # Третий прокси
```

##### 3. `get_random_proxy()`

Возвращает случайный прокси:

```python
proxy = get_random_proxy()  # Случайный прокси
```

##### 4. `get_proxy_count()`

Возвращает количество загруженных прокси:

```python
count = get_proxy_count()  # Например: 10
```

##### 5. `get_stats()`

Возвращает статистику:

```python
stats = get_stats()
# {
#     "total_loaded": 10,
#     "valid_proxies": 8,
#     "invalid_proxies": 2,
#     "current_proxy": "socks5://..."
# }
```

##### 6. `reset()`

Сбрасывает менеджер прокси:

```python
reset()  # Очищает все прокси
```

## 🔍 АЛГОРИТМ РОТАЦИИ

### Sequential (по умолчанию):

```
Прокси: [A, B, C, D, E]

Запрос 1 → A
Запрос 2 → B
Запрос 3 → C
Запрос 4 → D
Запрос 5 → E
Запрос 6 → A (начало цикла)
Запрос 7 → B
...
```

**Преимущества:**
- Равномерное распределение
- Предсказуемость
- Легко отследить

### Random (опционально):

```
Прокси: [A, B, C, D, E]

Запрос 1 → C
Запрос 2 → A
Запрос 3 → E
Запрос 4 → B
Запрос 5 → D
Запрос 6 → A
Запрос 7 → C
...
```

**Преимущества:**
- Меньше паттернов
- Сложнее отследить
- Лучше для анонимности
